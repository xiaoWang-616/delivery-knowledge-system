# 提交前检查

> 目标：提交系统能力和可复用知识，不把本地运行痕迹、smoke 产物和个人 Obsidian 状态混进去。

## 建议提交

- `apps/delivery-console/`：控制台前端和本地 runner。
- `scripts/`：启动、检查、停止脚本。
- `00-系统说明/`：系统定位和流程说明。
- `02-模块交付/`：模块资料包、设计与任务产物说明。
- `07-使用手册/`：使用流程和提交前检查。
- `11-规则库/`：AI 写代码、接口、样式、性能等规则。
- `README.md`、`.gitignore`。
- `08-项目实例/项目实例模板/`。
- 需要长期保留的真实项目沉淀，例如经过人工确认的 `08-项目实例/gil-business-web/`。

## 默认不提交

- `.obsidian/workspace.json`、`.obsidian/graph.json`：个人 Obsidian 工作区和图谱位置。
- `*.canvas`：临时画布，除非明确作为知识库资产。
- `apps/**/dist/`、`apps/**/node_modules/`：构建产物和依赖。
- `08-项目实例/delivery-console-smoke/`：runner smoke 测试产物。
- `08-项目实例/delivery-console/`：控制台自身接口 smoke 产物。
- `08-项目实例/**/runs/`：本地运行记录，默认是执行历史，不作为通用知识提交。

## 提交前命令

```bash
cd apps/delivery-console
pnpm run build
```

```bash
git status --short
```

## 判断规则

- 能复用的规则、模板、控制台代码：提交。
- 只属于本机窗口状态、测试过程、一次运行结果：不提交。
- 真实项目经验如果已经人工确认、能复用：可以提交到对应项目实例。
- 真实项目里包含敏感路径、接口返回、token、账号或业务隐私时，先脱敏再提交。
