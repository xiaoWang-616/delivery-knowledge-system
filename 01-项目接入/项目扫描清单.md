# 项目扫描清单

> 用途：给 AI 一个真实项目路径后，先按本清单扫描项目，不要直接写业务代码。

## 1. 基础文件扫描

需要读取：

- `package.json`
- `README.md`
- `tsconfig.json`
- 构建配置，例如 `vite.config.ts`、`webpack.config.js`、`next.config.js`
- lint 配置
- prettier 配置
- 环境变量示例，例如 `.env.example`、`.env.development`

输出：

```text
项目技术栈：
启动命令：
检查命令：
构建命令：
主要别名：
环境变量规则：
```

## 2. 目录结构扫描

优先查看：

- `src`
- `src/pages`
- `src/features`
- `src/components`
- `src/lib`
- `src/styles`
- `src/router`
- `src/layouts`

输出：

```text
页面目录：
业务模块目录：
通用组件目录：
接口封装目录：
样式目录：
工具函数目录：
```

## 3. 路由扫描

需要确认：

- 路由定义文件在哪里。
- 是否使用 React Router / Vue Router / Next Router 等。
- 是否有嵌套路由。
- 是否有动态路由。
- 是否有路由守卫。
- 浏览器返回是否依赖真实 URL。

输出：

```text
路由方案：
新增页面方式：
动态路由规则：
权限守卫：
返回路径规则：
```

## 4. 接口扫描

需要确认：

- 是否有统一 `httpClient`。
- API 文件放在哪里。
- 是否使用 TanStack Query / SWR / RTK Query。
- query key 如何组织。
- 错误处理在哪里做。
- token 注入在哪里做。

输出：

```text
接口封装方式：
页面是否允许直接请求：
服务端状态管理：
错误处理方式：
写操作刷新策略：
```

## 5. 组件扫描

需要确认：

- Button
- Input
- Select
- Table
- Pagination
- Dialog
- EmptyState
- Loading
- StatusBadge
- Tooltip
- Layout

输出：

```text
必须复用的组件：
可新增的组件类型：
组件风格：
组件尺寸规范：
```

## 6. 样式扫描

需要确认：

- 全局样式入口。
- 主题变量。
- Tailwind 配置。
- CSS Modules / SCSS / Less / CSS-in-JS。
- 设计 token。
- 字号、颜色、圆角、阴影、表格密度。

输出：

```text
样式方案：
主题色：
页面背景：
卡片样式：
表格密度：
按钮高度：
输入框高度：
```

## 7. 状态管理扫描

需要确认：

- 服务端状态。
- 全局状态。
- 表单状态。
- URL 状态。
- 页面离开时是否清空状态。

输出：

```text
服务端状态方案：
全局状态方案：
筛选状态规则：
表单状态规则：
```

## 8. 权限扫描

需要确认：

- 登录页。
- token 存储。
- 路由守卫。
- 菜单权限。
- 按钮权限。
- 角色切换。

输出：

```text
登录方式：
权限来源：
菜单权限：
按钮权限：
未登录处理：
无权限处理：
```

## 9. 测试能力扫描

需要确认：

- typecheck
- lint
- build
- unit test
- e2e test
- 浏览器手工验收

输出：

```text
必须通过的命令：
可选测试命令：
浏览器验收方式：
```
