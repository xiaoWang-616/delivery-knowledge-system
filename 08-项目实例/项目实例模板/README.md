# 项目实例模板

这是每个真实项目的工作区模板。

不要直接在这个模板里记录真实项目内容。接入新项目时，先复制一份目录，例如：

```text
08-项目实例/gil-business-web/
```

然后在复制后的目录里填写项目资料、模块状态、风险和交付记录。

## 推荐使用方式

1. 复制 `项目实例模板`。
2. 把目录改成真实项目名。
3. 填写 `项目总览.md`。
4. 把接口文档、Demo、旧项目、PRD 写入 `资料清单.md`。
5. 把要做的页面或功能写入 `模块清单.md`。
6. 用 `交付看板.md` 跟踪每个模块状态。
7. 用 `风险清单.md` 记录接口、样式、业务、测试风险。
8. 每个模块开始前，从 `09-模块看板` 复制模块状态、依赖、进度和问题看板。
9. 每个模块一轮开发完成后，按 `10-测试修复闭环` 做测试、问题汇总、人工确认和二次修复。
10. 模块完成后，把通用经验沉淀到 `11-规则库`，项目特有经验沉淀到本项目目录。

## 目录角色

| 文件 | 用途 |
| --- | --- |
| `项目总览.md` | 记录项目路径、技术栈、代码规则、全局约束 |
| `资料清单.md` | 记录接口文档、Demo、旧项目、PRD、截图等资料 |
| `模块清单.md` | 记录本项目要交付的模块 |
| `交付看板.md` | 记录模块推进状态 |
| `风险清单.md` | 记录 P0/P1/P2 风险和责任方 |
| `项目差异规则.md` | 记录只对当前项目生效的特殊规则 |

## 模块目录建议

每个模块建议单独建目录：

```text
模块名/
  模块状态卡片.md
  模块资料包.md
  模块依赖图.md
  模块依赖看板.md
  接口能力矩阵.md
  开发步骤卡片.md
  开发进度看板.md
  测试验收.md
  问题追踪看板.md
  规则沉淀.md
```

模块状态、依赖、进度、问题看板可以从：

```text
09-模块看板
```

复制得到。

## 项目特有规则

如果某条规则只对当前项目生效，不要直接写进通用规则库，建议新建：

```text
项目差异规则.md
项目接口规则.md
项目样式规则.md
项目性能规则.md
```

如果某条规则跨项目通用，再沉淀到：

```text
11-规则库
```

## 状态约定

| 状态 | 含义 |
| --- | --- |
| 待分析 | 资料已给齐，尚未分析 |
| 待确认 | AI 已输出方案，等待人工确认 |
| 开发中 | 正在写代码 |
| 待验收 | 已完成一轮代码，等待测试 |
| 待修复 | 已发现问题，等待二次修复 |
| 已完成 | 已验收并沉淀规则 |
