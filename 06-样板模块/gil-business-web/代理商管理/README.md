# 样板模块：gil-business-web / 代理商管理

这是模块项目交付知识库的第一个真实样板模块。它基于 `gil-business-web` 的代理商管理开发过程沉淀，展示一个模块从项目接入、模块资料包、依赖分析、接口能力矩阵、开发步骤、测试问题到规则沉淀应该如何记录。

## 样板价值

这个样板不是为了记录最终代码细节，而是为了沉淀“AI 写模块前应该知道什么、写完后应该记录什么”。

重点经验：

- 接口文档未定义的参数不能猜。
- 旧项目只能参考样式和交互，不能无脑照搬逻辑。
- Demo 样式优先级要提前确认。
- 三级省市区交互和接口真实能力可能不一致。
- 写操作要局部刷新相关 query，不能整页 reload。
- 第一次开发完成后，要把接口问题、样式问题、测试数据问题分级记录。

## 文件说明

- `模块资料包-示例.md`：代理商管理开工前应该整理出的资料包。
- `模块依赖图-示例.md`：代理商管理依赖 common、agent、opportunity 等能力的关系图。
- `接口能力矩阵-示例.md`：接口支持、限制、风险和降级策略。
- `开发步骤卡片-示例.md`：推荐开发拆分方式。
- `测试验收-示例.md`：本模块测试重点。
- `问题卡片-示例.md`：本模块遇到的典型问题。
- `规则沉淀-示例.md`：从本模块沉淀出的可复用规则。

## 使用方式

以后做新模块时，可以先看这个样板，再复制对应模板填写自己的模块资料。
