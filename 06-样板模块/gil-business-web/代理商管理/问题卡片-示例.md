# 问题卡片：代理商管理

## 问题 1：列表接口不支持区级筛选

| 项目 | 内容 |
| --- | --- |
| 等级 | P1 |
| 归属 | 后端接口能力 / 产品期望 |
| 状态 | 已降级处理 |

问题描述：

页面交互支持选择到 `北京市 / 市辖区 / 东城区`，但代理商列表接口只定义了 `provinceCode`，未定义 `cityCode/districtCode/regionCode`。

处理：

- 不传接口未定义参数。
- 选择到市/区后提示“当前接口不支持市/区精确筛选，会按所属省份查询”。

沉淀规则：

接口文档未定义的筛选参数，前端禁止猜测。
