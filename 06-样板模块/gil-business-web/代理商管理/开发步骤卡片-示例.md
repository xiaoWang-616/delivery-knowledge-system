# 开发步骤卡片：代理商管理

## 步骤 1：代理商列表

| 项目 | 内容 |
| --- | --- |
| 目标 | 完成列表、搜索、筛选、分页、详情入口 |
| 不做 | 不做详情写操作 |
| 关键接口 | `GET /agent-api/api/v1/admin/agents` |
| 验收 | 中文输入不失焦，筛选不整页刷新，分页局部刷新 |

风险：

- 区级筛选接口不支持，只能按省查询并提示。

## 步骤 2：代理商详情只读

| 项目 | 内容 |
| --- | --- |
| 目标 | 完成详情页布局、基础信息、销售团队、运营团队 tab |
| 不做 | 不提交写操作 |
| 关键接口 | 代理商详情、授权、协议、推广目标、成员 |
| 验收 | 浏览器 URL 可进入详情，tab 切换正常 |

## 步骤 3：代理商主档写操作

| 项目 | 内容 |
| --- | --- |
| 目标 | 新增、编辑基础资料、财务、邮寄、状态变更 |
| 不做 | 不处理协议和成员 |
| 关键接口 | POST/PUT/observe/activate/disable |
| 验收 | 成功后局部刷新，失败有错误提示 |

## 步骤 4：详情子资源

| 项目 | 内容 |
| --- | --- |
| 目标 | 授权区域、协议、推广目标、成员 |
| 不做 | 不猜未定义接口 |
| 关键接口 | authorizations、agreements、promotion-targets、members |
| 验收 | 弹窗可打开取消，提交后刷新详情 |

## 步骤 5：样式和测试收尾

| 项目 | 内容 |
| --- | --- |
| 目标 | 对齐 Demo，跑 typecheck/lint/build，浏览器点测 |
| 不做 | 不继续扩展新模块 |
| 验收 | 代码检查通过，问题形成卡片 |

## 总结

代理商管理是 L 级模块，不适合一次性写完。推荐按上述 5 步分阶段开发和验收。
