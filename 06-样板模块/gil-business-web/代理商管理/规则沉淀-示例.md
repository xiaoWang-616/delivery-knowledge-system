# 规则沉淀：代理商管理

## 规则 1：接口文档没定义的参数不能猜

来源问题：

- 代理商列表不支持 `cityCode/districtCode/regionCode`。

规则：

```text
如果接口文档没有定义某个参数，前端不能为了满足页面交互自行添加。
只能选择隐藏、降级、提示、或记录为后端待支持。
```

## 规则 2：旧项目只作为参考，不作为真相

来源问题：

- 旧项目有桥接脚本、mock、本地过滤和历史 bug。

规则：

```text
旧项目可以参考样式和交互，但接口路径、参数、响应字段、分页结构必须以接口文档为准。
```

## 规则 3：样式优先级必须提前确认

来源问题：

- 开发过程中样式参考源从旧项目切换到 Demo。

规则：

```text
模块开工前必须明确样式优先级。
推荐格式：设计图/Demo > 当前项目组件规范 > 旧项目样式。
```

## 规则 4：写操作必须局部刷新

来源问题：

- 企业级后台不能用整页刷新解决列表更新。

规则：

```text
新增、编辑、删除、状态变更成功后，只刷新受影响的 query。
禁止使用 window.location.reload()。
```

## 规则 5：第一次开发后必须形成问题卡片

来源问题：

- 接口问题、样式问题、测试数据问题如果只留在聊天里，后续模块会重复踩坑。

规则：

```text
模块第一次测试后，所有失败项和风险项都必须整理为问题卡片。
人工确认后再进入二次修复。
```
