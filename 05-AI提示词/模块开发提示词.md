# 模块开发提示词

```text
请先不要直接写代码。

请先阅读：
1. 项目画像
2. 模块资料包
3. 模块依赖图
4. 接口能力矩阵
5. 开发步骤卡片
6. 阻塞分级规则

然后先输出实现方案，方案必须包含：
- 本轮做什么。
- 本轮不做什么。
- 需要改哪些文件。
- 复用哪些组件。
- 使用哪些接口。
- 是否传了接口文档未定义的参数。
- 是否存在 P0/P1/P2 问题。
- 是否需要人工确认。
- 如何测试和验收。

我确认方案后，你再开始写代码。
写代码时必须遵守：
- 不猜接口字段。
- 不直接照搬旧项目 bug。
- 不整页 reload。
- 不前端全量拉数据再过滤。
- 写操作后局部刷新相关数据。
- 样式优先级按模块资料包执行。
```
