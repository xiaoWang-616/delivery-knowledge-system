# 模块项目交付知识库

这是一个专为“按模块交付项目”设计的知识库系统。它不绑定某一个业务项目，而是作为独立的交付方法库使用：每次接入真实项目路径，先生成项目画像，再整理模块资料包，最后指导 AI 按步骤开发、测试、修复和沉淀规则。

## 核心目标

- 让 AI 写代码前先理解项目结构、接口边界、样式规则和模块依赖。
- 把一个模块拆成可执行、可验收、可复盘的步骤卡片。
- 把接口问题、PRD 问题、样式问题、测试问题统一沉淀为问题卡片。
- 把每次踩坑沉淀成规则，让下一个模块少走弯路。
- 支持多个项目复用，只需要为每个项目生成独立项目画像。

## 当前阶段

第一期是 Markdown/Obsidian 可用的手动执行版，先把模板和流程跑通。

后续再逐步增加：

- 自动扫描项目路径。
- 自动生成项目画像。
- 自动生成模块依赖图。
- 自动生成测试用例和问题报告。
- 可视化看板与执行记录。

## 目录说明

- `00-系统说明`：系统定位、使用流程、阻塞分级。
- `01-项目接入`：项目画像、项目扫描、规则提取模板。
- `02-模块交付`：模块资料包、依赖图、开发步骤卡片。
- `03-测试验收`：功能测试、样式测试、接口测试、验收报告。
- `04-问题复盘`：问题卡片、二次修复、规则沉淀。
- `05-AI提示词`：项目分析、模块开发、代码审查、二次修复提示词。
- `06-样板模块`：真实项目模块样例，用来展示模板如何填写。
- `07-使用手册`：给实际使用者看的启动说明、模块启动流程。
- `08-项目实例`：每个真实项目的执行工作区模板和项目记录。
- `09-模块看板`：模块状态、依赖、开发进度、问题追踪的可视化模板。
- `10-测试修复闭环`：一轮开发后的测试、问题汇总、人工确认和二次修复流程。
- `11-规则库`：接口、样式、性能、AI 写代码和项目差异的通用规则。

## 使用方式

1. 用 Obsidian 打开本目录。
2. 先阅读 `07-使用手册/从0开始使用.md`。
3. 启动新模块时，把 `05-AI提示词/新模块启动总提示词.md` 复制给 AI。
4. 接入真实项目时，复制 `08-项目实例/项目实例模板`，创建项目自己的工作区。
5. 接入项目时，先填写项目画像和项目实例里的 `项目总览.md`。
6. AI 开发前必须先阅读 `11-规则库`。
7. 开发模块前，先填写模块资料包和模块依赖图。
8. 为模块创建模块状态卡片、依赖看板、开发进度看板和问题追踪看板。
9. AI 开发前必须先阅读项目画像、模块资料包、开发步骤卡片。
10. 第一次测试后，按 `10-测试修复闭环/测试执行说明.md` 执行验收。
11. 把问题写入问题卡片或问题追踪看板，并汇总到问题清单。
12. 人工确认问题后，用二次修复提示词启动修复。
13. 修复完成后，把可复用经验沉淀为规则库或项目特有规则。
