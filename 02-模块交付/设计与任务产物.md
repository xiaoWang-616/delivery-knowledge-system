# 设计与任务产物

> 用途：系统 AI 读取一次性资料后，先生成模块 design 和小任务队列，再把单个任务派发给写代码 AI。不要把整个模块交给写代码 AI 一次性生成。

## 目录结构

每个模块应生成：

```text
08-项目实例/{项目名}/{模块名}/设计与任务/
├── module-design.auto.md
├── task-queue.auto.json
├── task-01.prompt.auto.md
├── task-02.prompt.auto.md
├── task-03.prompt.auto.md
├── review-checklist.auto.md
└── progress.auto.md
```

## 文件职责

| 文件 | 给谁看 | 用途 |
| --- | --- | --- |
| `module-design.auto.md` | 系统 AI / 人 | 说明模块设计、边界、数据流、接口策略、组件拆分和验收策略。 |
| `task-queue.auto.json` | 系统 AI / 前端控制台 | 机器可读任务队列，记录任务状态、依赖、允许改动范围和验收标准。 |
| `task-xx.prompt.auto.md` | 写代码 AI | 单个小任务提示词。写代码 AI 每次只读取一个。 |
| `review-checklist.auto.md` | 系统 AI / 人 | 检查写代码 AI 结果是否越界、是否违反接口/样式/性能规则。 |
| `progress.auto.md` | 人 / 系统 AI | 展示当前进度、已完成任务、待修复任务和下一步。 |

## 模块 design 必须包含

- 模块目标和不做范围。
- 页面、路由、弹窗、抽屉和子资源范围。
- 接口文档来源和接口能力判断。
- 数据流、状态管理、缓存和局部刷新策略。
- 组件拆分和复用策略。
- 样式参考优先级。
- 测试策略和验收命令。
- 已知风险和待确认问题。

## 任务队列字段

`task-queue.auto.json` 中每个任务建议包含：

```json
{
  "id": "task-01",
  "title": "建立接口类型和 API 封装",
  "status": "pending",
  "dependsOn": [],
  "goal": "只完成接口类型、请求参数、响应类型和 API 函数。",
  "allowedFiles": ["src/features/example/api/**", "src/features/example/model/**"],
  "forbidden": ["不写页面 UI", "不接搜索筛选", "不写 mock 数据"],
  "inputs": ["模块上下文包.auto.md", "接口能力矩阵.auto.md"],
  "acceptance": ["类型命名清晰", "不直接 fetch", "接口路径来自接口文档"]
}
```

## 任务状态

| 状态 | 含义 |
| --- | --- |
| `pending` | 等待派发。 |
| `assigned` | 已派发给写代码 AI。 |
| `submitted` | 写代码 AI 已提交结果，等待系统 AI review。 |
| `reviewed` | 系统 AI 已检查，但还没决定是否完成。 |
| `needs-fix` | 检查未通过，需要生成修复任务。 |
| `done` | 任务完成，可以进入下一个任务。 |
| `blocked` | 当前任务不能继续，需要人工确认。 |

## 单任务 prompt 必须包含

- 当前任务编号和目标。
- 必须阅读的上下文文件。
- 允许修改的文件范围。
- 明确不做什么。
- 接口、样式、性能和状态管理规则。
- 验收方式。
- 完成后必须回报的内容。

写代码 AI 不应该看到完整模块就一把写完。它必须只完成当前任务。

## 系统 AI review 必须检查

- 是否越界修改非当前任务文件。
- 是否直接在页面里 fetch。
- 是否猜了接口文档没有定义的参数。
- 是否使用 `window.location.reload()`。
- 是否全量拉数据到前端过滤。
- 是否破坏项目已有风格。
- 是否缺少 loading / empty / error 状态。
- 是否需要运行 typecheck / lint / build。

## 推荐任务拆分顺序

```text
task-01 接口类型和 API 封装
task-02 静态页面结构和样式骨架
task-03 列表数据、分页和 loading/empty/error
task-04 搜索 debounce 和筛选
task-05 详情路由或详情抽屉
task-06 新增/编辑等写操作
task-07 权限、状态、边界和异常补齐
task-08 样式对齐和交互细节
task-09 命令验收、问题整理和修复任务
```

具体项目可以调整顺序，但不能把多个大型闭环塞进一个任务。
